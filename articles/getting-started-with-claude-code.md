## はじめに

Claude Codeは、ターミナル上で動くClaudeのCLIツールです。コードを書きながらClaudeに質問したり、バグ修正を依頼したり、コードレビューを頼んだりできます。

導入がシンプルで、Homebrewなら1コマンドで完了するため、その手順をまとめました。

## 準備するもの

- **OS**: macOS 10.15以降、Ubuntu 20.04以降、Windows 10以降
- **メモリ**: 4GB以上
- **ネット環境**: 必須
- **サブスクリプション**: Claude Pro/Max/Teams/Enterpriseプラン、またはConsoleアカウント

## インストール

Homebrewを使用している場合は、以下のコマンドでインストールできます。

```bash
brew install --cask claude-code
```

Homebrewを使っていない場合は、公式のインストールスクリプトを使用します。

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

どちらの方法も数秒で完了します。

## 起動

プロジェクトディレクトリに移動して、`claude`コマンドを実行します。

```bash
cd your-project
claude
```

初回起動時にログイン画面が表示されるので、Claude Pro/Max/Teams/Enterpriseのアカウント、またはConsoleアカウントでログインしてください。ログイン完了後、すぐに使用できます。

## 実際に使ってみる

起動したら、自然な日本語で話しかけることができます。

```
> このコードの問題点を教えて
```

ファイルを指定して質問することもできます。

```
> src/index.tsのバグを修正して
```

コードレビューも依頼できます。

```
> 最後のコミットをレビューして
```

会話しながらコードを修正していくことができ、VSCodeやCursorのような感覚で使えますが、ターミナル上で完結します。

## 動作確認

インストールが正しく完了しているか確認したい場合は、以下のコマンドを実行します。

```bash
claude doctor
```

エラーが表示されなければ、正常にインストールされています。

## アップデート

### Homebrewでインストールした場合

手動でアップデートする必要があります。

```bash
brew upgrade claude-code
```

### 公式スクリプトでインストールした場合

自動更新が有効になっているため、通常は何もする必要がありません。起動時やバックグラウンドで自動的に更新され、次回起動時に適用されます。

手動で更新を確認したい場合は、以下のコマンドを使用できます。

```bash
claude update
```

## よくあるエラー

### ログインできない

まず、`claude doctor`を実行して状態を確認します。

```bash
claude doctor
```

ネットワーク接続の問題か、サブスクリプションプランの契約状況を確認してください。

### コマンドが見つからない

パスが正しく設定されていない可能性があります。まず、ターミナルを再起動してみてください。

```bash
# ターミナルを再起動
exit
```

それでも解決しない場合は、再インストールを試してください。

```bash
brew reinstall claude-code
```

## まとめ

Claude Codeは、Homebrewなら1コマンドで導入でき、プロジェクトディレクトリで`claude`を実行すればすぐに使い始められます。

ターミナルから離れずにコードレビューやバグ修正ができるのは、実際に使ってみると想像以上に便利です。特に、エラーメッセージをそのまま貼り付けて修正を依頼できる点が魅力的です。

Claude Pro/Max/Teams/Enterpriseプラン、またはConsoleアカウントをお持ちであれば、追加料金なしで利用できるので、ぜひ試してみてください。
