# WCAG 2.1準拠チェックリスト - 実装の確認

## チェックリストの使い方

このチェックリストは、WCAG 2.1 Level AA準拠を確認するための実践的なガイドです。各項目について、実装時と公開前に確認してください。

### チェック方法

- ✅: 実装済み・準拠している
- 🔄: 一部実装・改善の余地あり
- ❌: 未実装・準拠していない
- N/A: 該当なし

## Perceivable（知覚可能）

### 1.1 テキストによる代替

#### 1.1.1 非テキストコンテンツ（Level A）

- すべての画像にalt属性がある
- 装飾的な画像はalt=""またはrole="presentation"
- アイコンボタンにaria-labelがある
- キャプチャ（CAPTCHA）に代替手段がある
- 複雑な画像（グラフ等）に詳細な説明がある

```tsx
// チェック方法
// Chrome DevTools > Elements > Accessibility Tree
// または axe DevTools で確認
```

### 1.2 時間依存メディア

#### 1.2.1 音声のみ及び映像のみ（Level A）

- 音声のみコンテンツにトランスクリプトがある
- 映像のみコンテンツに音声解説またはトランスクリプトがある

#### 1.2.2 キャプション（Level A）

- すべての動画に字幕がある
- WebVTTまたは同等の形式で提供

#### 1.2.3 音声解説またはメディアの代替（Level A）

- 動画に音声解説またはトランスクリプトがある

#### 1.2.5 音声解説（Level AA）

- すべての動画に音声解説がある

### 1.3 適応可能

#### 1.3.1 情報及び関係性（Level A）

- セマンティックHTML要素を使用（header、nav、main、article、aside、footer）
- 見出し階層が論理的（h1→h2→h3、スキップなし）
- リストにul/ol/li要素を使用
- データテーブルにth、caption、scopeを使用
- フォームにlabel要素を使用
- 関連するフォーム要素をfieldset/legendでグループ化

```tsx
// チェック方法
const checkHeadingHierarchy = () => {
  const headings = Array.from(document.querySelectorAll('h1, h2, h3, h4, h5, h6'))
  headings.forEach((heading, index) => {
    const level = parseInt(heading.tagName[1])
    const prevLevel = index > 0 ? parseInt(headings[index - 1].tagName[1]) : 0
    if (level - prevLevel > 1) {
      console.warn(`見出しがスキップされています: ${headings[index - 1].textContent} (h${prevLevel}) → ${heading.textContent} (h${level})`)
    }
  })
}
```

#### 1.3.2 意味のある順序（Level A）

- DOM順序が視覚的な順序と一致
- CSSのorder、flex-direction、grid等で順序を変更していない
- tabIndexで順序を変更していない（0または-1のみ使用）

#### 1.3.3 感覚的な特徴（Level A）

- 色だけで情報を伝えていない（アイコン、テキストも使用）
- 位置だけで指示していない（「右側のボタン」ではなく「送信ボタン」）
- サイズ・形だけで区別していない

#### 1.3.4 表示の向き（Level AA）

- 縦向き、横向きの両方で表示可能
- 向きを固定していない

#### 1.3.5 入力目的の特定（Level AA）

- フォーム入力にautocomplete属性がある（name、email、tel等）

### 1.4 判別可能

#### 1.4.1 色の使用（Level A）

- 色だけで情報を伝えていない
- リンクが色以外でも判別できる（下線、アイコン等）
- エラーメッセージが色とテキストで表示

#### 1.4.3 コントラスト（Level AA）

- 通常テキスト: 4.5:1以上
- 大きなテキスト（18pt以上または14pt太字以上）: 3:1以上
- Chrome DevTools > Lighthouseで確認

```tsx
// チェック方法
// Chrome DevTools > Lighthouse > Accessibility
// または WebAIM Contrast Checker
```

#### 1.4.4 テキストのサイズ変更（Level AA）

- 200%拡大しても情報が失われない
- 固定ピクセル値を避け、rem/emを使用

#### 1.4.10 リフロー（Level AA）

- 320pxの幅で水平スクロールが発生しない
- レスポンシブデザインで実装

#### 1.4.11 非テキストコントラスト（Level AA）

- UI要素（ボタン、フォーム等）: 3:1以上
- フォーカスインジケーター: 3:1以上

#### 1.4.12 テキストの間隔（Level AA）

- 行間1.5倍、段落間2倍でも表示可能
- letter-spacing、word-spacingの調整に対応

#### 1.4.13 ホバーまたはフォーカスで表示されるコンテンツ（Level AA）

- ツールチップがEscで閉じられる
- ツールチップ自体にホバーできる
- ツールチップが自動的に消えない（または設定可能）

## Operable（操作可能）

### 2.1 キーボードアクセシブル

#### 2.1.1 キーボード（Level A）

- 全ての機能がキーボードで操作可能
- Tabキーで全要素に到達できる
- Enter/Spaceでボタンが動作
- カスタム要素にキーボードイベントハンドラがある

```tsx
// チェック方法
// マウスを使わずにTabキーだけで全ての操作を試す
```

#### 2.1.2 キーボードトラップなし（Level A）

- フォーカスが閉じ込められない
- モーダルでEscapeキーで閉じられる
- フォーカストラップは意図的な場合のみ（モーダル等）

#### 2.1.4 文字キーのショートカット（Level A）

- 文字キーショートカットに無効化・再割り当て機能がある

### 2.2 十分な時間

#### 2.2.1 タイミング調整可能（Level A）

- タイムアウトに警告と延長機能がある
- 制限時間を無効化できる

#### 2.2.2 一時停止、停止、非表示（Level A）

- 自動再生コンテンツに停止ボタンがある
- カルーセルに一時停止ボタンがある
- 5秒以上自動更新されるコンテンツに停止機能がある

### 2.3 発作と身体的反応

#### 2.3.1 3回の閃光、又は閾値以下（Level A）

- 1秒間に3回以上点滅するコンテンツがない
- prefers-reduced-motionに対応

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}
```

### 2.4 ナビゲーション可能

#### 2.4.1 ブロックスキップ（Level A）

- スキップリンクがある
- スキップリンクがフォーカス時に表示される

#### 2.4.2 ページタイトル（Level A）

- 全ページに内容を説明するtitle要素がある
- titleが一意で説明的

#### 2.4.3 フォーカス順序（Level A）

- フォーカス順序が論理的
- tabIndexで順序を変更していない

#### 2.4.4 リンクの目的（Level A）

- リンクテキストが目的を説明している
- 「こちら」「詳細」等の不明確なテキストを避ける
- 必要に応じてaria-labelで補足

#### 2.4.5 複数の手段（Level AA）

- サイト内検索がある
- サイトマップがある
- ナビゲーションメニューがある

#### 2.4.6 見出し及びラベル（Level AA）

- 見出しが内容を説明している
- フォームラベルが目的を説明している

#### 2.4.7 フォーカスの可視化（Level AA）

- フォーカスインジケーターが視認できる
- outline: noneを使用していない（またはカスタムフォーカススタイル）

### 2.5 入力モダリティ

#### 2.5.1 ポインタジェスチャー（Level A）

- 複雑なジェスチャに代替手段がある
- スワイプにボタンでの代替がある

#### 2.5.2 ポインタのキャンセル（Level A）

- クリック操作がmouseupで発火（キャンセル可能）

#### 2.5.3 ラベルを含む名前（Level A）

- 視覚的なラベルとaria-labelが一致
- aria-labelが視覚的なテキストを含む

#### 2.5.4 動きによる起動（Level A）

- デバイスの動きによる機能に代替手段がある

## Understandable（理解可能）

### 3.1 読みやすさ

#### 3.1.1 ページの言語（Level A）

- html要素にlang属性がある

```html
<html lang="ja">
```

#### 3.1.2 一部分の言語（Level AA）

- 他の言語部分にlang属性がある

```html
<p>この技術は<span lang="en">WCAG 2.1</span>に準拠しています。</p>
```

### 3.2 予測可能

#### 3.2.1 フォーカス時（Level A）

- フォーカスだけでコンテキストが変化しない

#### 3.2.2 入力時（Level A）

- 入力だけで自動送信されない
- 変更には明示的な送信ボタンがある

#### 3.2.3 一貫したナビゲーション（Level AA）

- ナビゲーションが全ページで同じ順序
- 共通コンポーネントが一貫している

#### 3.2.4 一貫した識別性（Level AA）

- 同じ機能が同じラベル・アイコンで識別される

### 3.3 入力支援

#### 3.3.1 エラーの特定（Level A）

- エラーがテキストで説明される
- エラー箇所がaria-invalidで識別される
- エラーメッセージがaria-describedbyで関連付けられる

#### 3.3.2 ラベル又は説明（Level A）

- 全てのフォーム入力にラベルがある
- 必須項目が明示されている
- 入力形式が説明されている

#### 3.3.3 エラー修正の提案（Level AA）

- エラー時に修正案が提示される
- 例: メールアドレスのスペルチェック

#### 3.3.4 エラー回避（Level AA）

- 重要な操作に確認ステップがある
- 削除操作に確認ダイアログがある
- フォーム送信前に確認画面がある

## Robust（堅牢）

### 4.1 互換性

#### 4.1.2 名前（name）、役割（role）及び値（value）（Level A）

- 全UI要素にアクセシブルな名前がある
- カスタム要素に適切なroleがある
- ステート（aria-expanded、aria-pressed等）が正しく更新される

#### 4.1.3 ステータスメッセージ（Level AA）

- ステータス変更がaria-liveで通知される
- role="status"またはaria-live="polite"を使用
- エラーはaria-live="assertive"

## まとめ

WCAG 2.1準拠には、自動テストと手動テストの組み合わせが重要です。このチェックリストを活用して、段階的にアクセシビリティを向上させてください。

### 優先順位

1. **Level A**: 必須の基準、まず対応
2. **Level AA**: 推奨の基準、次に対応
3. **継続的改善**: CI/CDに統合、自動チェック
